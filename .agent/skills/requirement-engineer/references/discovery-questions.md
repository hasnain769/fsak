# Discovery Questions Reference

Structured question bank for the discovery phase. Select questions based on project context.

## Business Context

**Why build this?**
- What problem does this solve? Who experiences this problem today?
- What's the cost of NOT solving this problem?
- Who are your competitors? How will this be different?

**Target Audience**
- Who is the primary user? (Role, demographics, tech savviness)
- Are there secondary users with different needs?
- How will users discover/access this product?

## Functional Scope

**Core Value**
- If users could only do ONE thing, what would it be?
- What's the minimum feature set for a useful product?
- What features are "nice to have" vs. dealbreakers?

**User Journeys**
- Walk me through a typical user session from start to finish
- What triggers a user to open this app?
- What does success look like for a user?

**MVP Boundaries**
- What can we explicitly exclude from v1?
- What's the timeline for launch?
- What would make you consider this project successful?

## Technical Constraints

**Platform & Stack**
- Web, mobile, desktop, or all?
- Any required technologies? (e.g., "Must use React", "Must integrate with Salesforce")
- Any forbidden technologies? (e.g., "No vendor lock-in")

**Integrations**
- What existing systems must this connect to?
- What data needs to flow in/out?
- Are there APIs we need to consume or provide?

**Infrastructure**
- Cloud preference? (AWS, GCP, Azure, self-hosted)
- Any compliance requirements? (GDPR, HIPAA, SOC2)
- Expected data volumes/retention needs?

## Non-Functional Requirements

**Performance**
- Expected concurrent users?
- Acceptable response times?
- Any offline/low-connectivity requirements?

**Security**
- Authentication needs? (SSO, MFA, social login)
- Authorization model? (Roles, permissions, multi-tenancy)
- Sensitive data handling requirements?

**Scalability**
- Growth expectations? (users, data, transactions)
- Seasonal/peak load patterns?
- Geographic distribution?

## Project Constraints

**Resources**
- Team size and composition?
- Budget constraints?
- Existing codebase or greenfield?

**Timeline**
- Hard deadlines? (Launch date, demo date, regulatory)
- Phased rollout planned?
- What's flexible vs. fixed?

---

## Probing Techniques

| Situation | Technique |
|-----------|-----------|
| Vague answer | "Can you give me a specific example?" |
| Too broad | "If you had to pick just one, which would it be?" |
| Conflicting requirements | "These seem to conflict—which takes priority?" |
| Assumed knowledge | "I want to make sure I understand—can you explain X?" |
| Missing stakeholder | "Who else should weigh in on this decision?" |
